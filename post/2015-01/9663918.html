Chapter 4 : Divide-and-Conquer


분할하고 정복하기 3단계

  1. 분할하라 : 문제를 보다 더 작은 하위 문제로 분할하라

  2. 정복하라 : 재귀적으로 하위 문제를 풀어 정복하라. 그러나 하위 문제의 크기가 작다면 직접적인 방법으로 하위 문제를 풀어라.

  3. 조합하라 : 하위 문제에 대한 답을 원래 문제에 대한 답으로 조합하라.

  • recursive case : 하위 문제가 충분히 커서 재귀할 수 있는 경우

  • base case ( ="bottoms out") : 하위 문제가 충분히 작아 더 이상 재귀할 수 없는 경우


Recurrences

  • 재귀는 작은 입력에 대한 값에 관한 함수를 설명하는 등식 또는 부등식이다.



  • 재귀는 항상 같은 크기를 가지진 않는다. 다양한 형태를 취할 수 있다.

    ① T(n) = T(2n/3) + T(n/3) + Θ(n)

    ② T(n) = T(n-1) + Θ(1)               // 선형탐색 식. Θ(1)은 재귀시 걸리는 시간이다.

  • 재귀는 등식이 아닌 부등식일 수도 있다.

    ex) T(n) ≤ 2T(n/2) + Θ(n)   // Θ-표기 사용

         T(n) ≥ 2T(n/2) + Θ(n)   // Ω-표기 사용

6-1Building a heap using insertion

We can build a heap by repeatedly calling MAX-HEAP-INSERT to insert the elements into the heap. Consider the following variation on the BUILD-MAX-HEAP procedure:

BUILD-MAX-HEAP'(A)
A.heap-size = 1
for i=2 to A.length
   MAX-HEAP-INSERT(A,A[i])

a. Do the procedures BUILD-MAX-HEAP and BUILD-MAX-HEAP' always create the same heap when run on the same input array? Prove that they do, or provide a counterexample.

b. Show that in the worst case, BUILD-MAX-HEAP' requires Θ(n lg n) time to build an n-element heap.


  1. 아니다. 서로 다른 결과를 낳는다. (아래 코드 참조)
    Heap builds for:1, 2, 3, 4, 5, 6
    -------------------------------------------------
    BUILD-MAX-HEAP: 6, 5, 3, 4, 2, 1
    BUILD-MAX-HEAP':6, 4, 5, 1, 3, 2

  2. MAX-HEAP-INSERT 는 worst-case에서 Θ(lg n) 실행시간을 가지며, n-1번 호출된다. 호출될 때마다 Θ(lg n) 시간복잡도가 수행되므로 식은 다음과 같다.
    $$\sum_{i=2}^{n}\lg{i} = \lg(n!) = \Theta(n\lg{n})$$
    (참조 : exercise 3.2.3)

Java code
class Heap {
int heap_size;
int length;
int[] elements;

Heap(int[] elements) {
this.elements = elements;
length = elements.length;
}

public void increment() {
int[] copy = new int[++length];
System.arraycopy(elements, 0, copy, 0, length-1);
elements = copy;
}
}

class HeapMain
{
public static void main(String[] args)
{
int[] num = {1,2,3,4,5,6};

Heap heap = new Heap(num);

BuildMaxHeap(heap);
for(int idx : heap.elements)
System.out.print(idx + " ");

System.out.println();

num = new int[]{1,2,3,4,5,6};

heap = new Heap(num);
BuildMaxHeap2(heap);
for(int idx : heap.elements)
System.out.print(idx + " ");
}

static void BuildMaxHeap(Heap h) {
h.heap_size = h.length;
for(int i = (h.heap_size-1)/2; i >= 0; i--)
MaxHeapify(h, i);
}

static void BuildMaxHeap2(Heap h) {
h.heap_size = 1;
for(int i=1; i < h.length; i++)
MaxHeapInsert(h, h.elements[i]);
}

static void MaxHeapify(Heap h, int i) {
int left = LEFT(i);
int right = RIGHT(i);
int largest = i;

if (left < h.heap_size && h.elements[left] > h.elements[i])
largest = left;

if (right < h.heap_size && h.elements[right] > h.elements[i])
largest = right;

if (largest != i) {
int temp = h.elements[i];
h.elements[i] = h.elements[largest];
h.elements[largest] = temp;
MaxHeapify(h, largest);
}
}

static void HeapIncreaseKey(Heap h, int i, int key) {
if(key < h.elements[i]) {
System.out.println("new key is smaller than current key");
return;
}

for (int parent; i > 0 && h.elements[parent = PARENT(i)] < key; i = parent)
h.elements[i] = h.elements[parent];

h.elements[i] = key;
}

static void MaxHeapInsert(Heap h, int key) {
if(++h.heap_size > h.length) h.increment();
h.elements[h.heap_size-1] = Integer.MIN_VALUE;

HeapIncreaseKey(h, h.heap_size-1, key);
}

static int PARENT(int i) {
return (i-1) / 2;
}

static int LEFT(int i) {
return (2*i + 1);
}

static int RIGHT(int i) {
return (2*i + 2);
}
}





6-2Analysis of d-ary heaps

A d-ary heap is like a binary heap, but (with one possible exception) non-leaf nodes have d children instead of 2 children.

a. How would you represent a d-ary heap in an array?

b. What is the height of a d-ary heap of n elements in terms of n and d?

c. Give an efficient implementation of EXTRACT-MAX in a d-ary max-heap. Analyze its running time in terms of d and n.

d. Give an efficient implementation of INSERT in a d-ary max-heap. Analyze its running time in terms of d and n.

e. Give an efficient implementation of INCREASE-KEY(A,i,k), which flags an error if k < A[i], but otherwise sets A[i] = k and then updates the d-ary max-heap structure appropriately. Analyze its running time in terms of d and n.


※ 인덱스는 1부터 시작한다.
  1. d-ary heap의 k번째 자식 노드 점화식
    $$di + k - (d-1)$$
    d-ary heap의 부모 노드 점화식
    $$\left \lceil \frac{(i-1)}{d} \right \rceil $$

  2. height : $ \log_{d} n$

  3. binary heap의 HEAP-EXTRACT-MAX와 같다. MAX-HEAPIFY에서 매번 자식 노드의 개수인 d만큼 비교하므로 log 앞에 d가 붙는다.

    running time : $O(d\log_{d}n)$

    HEAP-EXTRACT-MAX(A)
    if A.heap-size < 1
       error "heap underflow"
    max = A[1]
    A[1] = A[A.heap-size]
    A.heap-size = A.heap-size - 1
    MAX-HEAPIFY(A,1)
    return max

  4. binary heap과 같다.

    running time : $O(\log_{d}n)$

    MAX-HEAP-INSERT(A, key)
    A.heap-size = A.heap-size + 1
    A[A.heap-size] = -∞
    HEAP-INCREASE-KEY(A, A.heap-size, key)

  5. 부모노드만 선택해서 비교하므로 log 앞에 d 계수가 붙지 않는다.

    running time : $O(\log_{d}n)$

    HEAP-INCREASE-KEY(A,i,key)
    if key < A[i]
       error "new key is smaller than current key"
    A[i] = key
    while i > 1 and A[PARENT(i)] < A[i]
       exchange A[i] with A[PARENT(i)]
       i = PARENT(i)





6-3 Young tableaus

An m x n Young tableau is an m x n matrix such that the entries of each row are in sorted order from left to right and the entries of each column are in sorted order form top to bottom. Some of the entries of a Young tableau may be ∞, which we treat as nonexistent elements. Thus, a Young tableau can be used to hold r ≤ mn finite numbers.

a. Draw a 4 x 4 Young tableau containing the elements {9, 16, 3, 2, 4, 8, 5, 14, 12}.

b. Argue that an m x n Young tableau Y is empty if Y[1,1] = ∞. Argue that Y is full (contains mn elements) if Y[m,n] < ∞. 

c. Give an algorithm to implement EXTRACT-MIN on a nonempty m x n Young tableau that runs in O(m + n) time. Your algorithm should use a recursive subroutine that solves an m x n problem by recursively solving either an (m-1) x n or an m x (n-1) subproblem. (Hint: Think about MAX-HEAPIFY.) Define T(p), where p = m + n, to be the maximum running time of EXTRACT-MIN on any m x n Young tableau. Give and solve a recurrence for T(p) that yields the O(m + n) time bound.

d. Show how to insert a new element into a nonfull m x n Young tableau in O(m + n) time.

e. Using no other sorting method as a subroutine, show how to use an n x n Young tableau to sort n2 numbers in O(n3) time.

f. Give an O(m + n)-time algorithm to determine whether a given number is stored in a given m x n Young tableau.

  1. 아래 답 외에 다양한 Young tableau가 존재한다.
    \[\begin{matrix} 2& 3& 4& 5\\ 8& 9& 12& 14\\ 16& \infty& \infty& \infty\\ \infty& \infty& \infty& \infty \end{matrix}\]

  2. ¹ ∞가 Y[1,1]에 위치하면 모든 요소는 ∞와 같거나 커야 한다. ∞보다 큰 것은 없으므로 모든 요소는 ∞를 가진다. ∞요소를 존재하지 않는 요소라 정의했으므로 Y는 비게 된다.

    ² Y[m,n] < ∞ 비교문은  ∞보다 작은 값으로 모든 요소가 채워져 있다는 걸 뜻한다.


  3. EXTRACT-MIN(Y)
    min = Y[1,1]
    Y[1,1] = ∞
    MIN-MAKE(Y,1,1)
    return min

    MIN-MAKE(Y,i,j)
    if i == m and j == n
       return;
    Y[i,j] = MIN(Y[i,j], Y[i+1,j], Y[i,j+1])
    if i < m and Y[i,j] == Y[i+1,j]
       MIN-MAKE(Y,i+1,j)
    else if j < n and Y[i,j] == Y[i,j+1]
       MIN-MAKE(Y,i,j+1)
    $$T(p) = T(p-1) + O(1) = ... = O(p)$$

  4. c의 MIN-MAKE을 응용하면 쉽게 풀 수 있다.

    INSERT(Y,key)
    Y[m,n] = key
    MIN-MAKE-INVERSE(Y,m,n)

    MIN-MAKE-INVERSE(Y,i,j)
    if i == 1 and j == 1
       return;
    Y[i,j] = MAX(Y[i,j], Y[i-1,j], Y[i,j-1])
    if i > 1 and Y[i,j] == Y[i-1, j]
       MIN-MAKE-INVERSE(Y,i-1,j)
    else if j > 1 and Y[i,j] == Y[i,j-1]
       MIN-MAKE-INVERSE(Y,i,j-1)

  5. 배열 A를 가지고 Young tableau를 만드는데는 O(n + n)이 걸린다. 모든 요소를 삽입하는데 걸리는 시간은 O(n2) 이다. 따라서, 총 실행시간은 O(n) * O(n2) = O(n3) 이다.


  6. FIND-YOUNG(Y,key)
    if key >= Y[1,1] and key <= Y[m,n]
       return true
    else
       return false
